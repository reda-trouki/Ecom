# Screenshots Directory

This directory contains all the screenshots for the README documentation.

## Structure

```
docs/images/
├── client/              # Customer application screenshots
│   ├── homepage.png     # Homepage with hero section
│   ├── products.png     # Product catalog page
│   ├── product-detail.png # Individual product page
│   ├── cart.png         # Shopping cart page
│   ├── checkout.png     # Checkout process
│   └── orders.png       # Order history page
└── admin/               # Admin dashboard screenshots
    ├── dashboard.png    # Main analytics dashboard
    ├── products.png     # Product management interface
    ├── orders.png       # Order management system
    └── analytics.png    # Analytics and reports
```

## Guidelines for Screenshots

### Image Requirements
- **Format**: PNG (preferred) or JPG
- **Resolution**: 1920x1080 or higher
- **Quality**: High quality, clear and readable text
- **Browser**: Use latest Chrome/Firefox with clean interface

### Naming Convention
- Use lowercase with hyphens for separation
- Be descriptive but concise
- Follow the structure outlined above

### Content Guidelines
- Show real data (not Lorem ipsum)
- Include realistic product images
- Demonstrate key features and functionality
- Ensure UI is in its best state (no error messages unless showcasing error handling)

### Taking Screenshots
1. Use full viewport screenshots
2. Ensure consistent browser window size
3. Remove browser UI elements if possible
4. Show meaningful data and interactions
5. Include both desktop and mobile views where relevant

## Adding New Screenshots

1. Place screenshots in the appropriate directory (`client/` or `admin/`)
2. Update the README.md if adding new screenshot categories
3. Ensure file names match those referenced in the main README
4. Optimize images for web (compress without losing quality)

## Tools for Screenshots

- **macOS**: Cmd + Shift + 4 (area) or Cmd + Shift + 3 (full screen)
- **Windows**: Windows + Shift + S (Snipping Tool) or PrtScn
- **Browser Extensions**: 
  - Awesome Screenshot
  - Full Page Screen Capture
  - Lightshot

## Image Optimization

After taking screenshots, consider optimizing them:
- Use [TinyPNG](https://tinypng.com/) for compression
- [ImageOptim](https://imageoptim.com/) for Mac users
- [Squoosh](https://squoosh.app/) for web-based optimization

Remember to keep file sizes reasonable while maintaining quality for a good documentation experience.
